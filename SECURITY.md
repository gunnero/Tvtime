# Security policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Send a private report to `aleksandar.dimovski@me.com` with the affected component, impact, reproduction steps, and any suggested mitigation. Remove credentials, personal data, private URLs, and infrastructure details from the report unless they are strictly necessary and shared through an agreed secure channel.

You can expect acknowledgement within five business days, an initial assessment after reproduction, and coordinated remediation and disclosure when appropriate. Please allow a reasonable remediation window before public discussion.

## Supported versions

MediaHub is in active pre-release development. Security fixes target the latest commit on the default branch and the most recent release candidate. Older development snapshots are not supported.

## Disclosure boundaries

- Never publish credentials, tokens, private keys, user exports, or personal viewing data.
- Never publish private hosts, service ports, server paths, deployment topology, or rollback locations.
- Use minimal synthetic test data when demonstrating a report.
- Public vulnerability reports and proof-of-concept exploitation against live systems are not accepted.
